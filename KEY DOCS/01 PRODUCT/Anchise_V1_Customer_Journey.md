# Anchise V1 Customer Journey

## User-Facing Journey

The customer journey should be short and legible:

1. Add source
2. Connect account or choose local folders
3. Review what is there
4. Choose what to bring into Anchise
5. Import to Anchise cloud
6. Organize and protect

`Review what is there` is the user-facing label for the inventory step.

## Step 3: Review What Is There

Before any acquisition begins, Anchise should show:

- categories available
- estimated source size
- estimated net-new size after comparing against Anchise storage
- available Anchise storage
- a simple fit result: fits, likely exceeds, exceeds, or unknown
- whether future refreshes can stay incremental

This is the moment Anchise proves it is a control system rather than a downloader.

## Why This Step Matters

- It keeps the experience simple because the user sees the decision before the cost.
- It keeps infrastructure cost low because only lightweight metadata is collected first.
- It keeps trust high because the system says what will fit before it starts moving large amounts of content.

## V1 Language To Prefer

- Add source
- Connect account
- Review what is there
- Bring into Anchise
- Recent import
- Location history
- Locked folder

## V1 Language To Avoid

- Scan inventory as a primary label
- Latest downloaded blob as a primary user-facing label
- Delete everything
- Full account extraction guaranteed
